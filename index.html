RODEO is a vendor-agnostic evaluation platform that lets robotics teams benchmark policies, simulators, and hardware through a single unified interface.

Get Started
Why RODEO

Stop rebuilding evaluation pipelines

Robotics teams waste months wiring up simulators, policy frameworks, and hardware SDKs. RODEO provides the connective tissue so you can focus on what matters: building better robots.

🔄

Swap anything, change nothing

Move between MuJoCo, Isaac Sim, PyBullet, or real hardware without touching your evaluation code. Vendor lock-in disappears.

📊

Apples-to-apples comparison

Run the same policy across different simulators and hardware platforms with consistent metrics. Make decisions based on data, not intuition.

⚡

From sim to real in minutes

Unified data formats and hardware bridges mean policies trained in simulation transfer to physical robots through the same API.

🔌

Plugin-first architecture

Every backend, policy framework, and hardware bridge is a plugin. Add your own with a single entry point — no forks, no patches.

🏢

Lightweight core, optional everything

The core depends only on NumPy, PyArrow, Pydantic, and PyYAML. Install only the backends you need. No 10 GB base install.

🚀

Production-grade services

FastAPI microservices for simulation, learning, and hardware. Docker-ready. PostgreSQL-backed. Built for teams, not just notebooks.

Technical Features

Built for robotics engineers

Protocol-driven design with runtime checking, type safety, and clean separation of concerns across every layer.

Runtime Protocols

All adapters implement runtime-checkable protocols defined in rodeo.contracts. Type errors surface immediately, not in production.

Generic Registry

Registry[T] discovers plugins via Python entry points. Lazy loading, type-safe, with automatic validation on registration.

Canonical Data Schemas

Pydantic v2 schemas for observations, actions, episodes, and metrics. One format across all adapters.

6 Data Adapters

Import and export between native, LeRobot, RLDS, Open X-Embodiment, ROS bags, and CSV. Lossless round-trip conversion.

VLA & Foundation Models

First-class support for Vision-Language-Action models, including HuggingFace Transformers integration and API-based remote inference.

World Model Adapters

Plug in Cosmos, UniSim, or Genie2 for learned world models. Evaluate policies in imagination before committing to sim or hardware.

Microservice Architecture

Simulation, learning, and hardware services run independently. Scale each layer based on your workload.

CLI + API

Full CLI for scripting and automation. REST API for integration with CI/CD, dashboards, and external tools.

Ecosystem

Integrations out of the box

30+ adapters across simulation, policy, data, hardware, and world models. Each one is a plugin you can extend or replace.
